Placement Rules â€” Six Laws of File Governance
Rule 01
One Home Per File
Every file resides in exactly one location. Link to it from other platforms rather than copying. If the same file exists in two locations, one copy should be removed.
Rule 02
OneDrive Is for Active Drafts, Not Long-Term Storage
Files in OneDrive are in progress â€” drafts, iterative documents, and active development. Once finalized, they migrate to SharePoint for reference, to Teams for team use, or are shared via Outlook link.
Rule 03
If the Team Needs It Daily, It Belongs in Teams Files
Timesheets, schedules, log books, templates, handoff notes â€” anything an assistant touches during a shift. Collaborative from birth. No OneDrive detour needed.
Rule 04
SharePoint Is the Library, Not the Workshop
Finalized SOPs, onboarding guides, training materials, compliance docs. Read-only for staff. Only the Lab Manager publishes. Version control and SOP integrity depend on this boundary.
Rule 05
The Continuity Check
If a colleague would need this file to perform their duties in your absence, it should not remain in personal OneDrive as its permanent home. Move it to Teams Files or SharePoint.
Rule 06
Share via Link, Not Attachment
When sharing files externally via Outlook, send a OneDrive or SharePoint link â€” not an attachment. This preserves version control and prevents outdated copies from circulating.
